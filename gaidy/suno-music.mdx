<Note>
Полная документация API: [Генерация медиа](/api-reference/media/create)
</Note>

## Операции над генерацией

Для модели `suno/generate` доступны операции через `POST /api/v1/media/:id/operations`:

| Операция | Описание | Стоимость |
|----------|----------|-----------|
| `extend` | Продление трека от указанной секунды | 15 ₽ |
| `wav` | Конвертация в WAV | 1 ₽ |
| `vocal_removal` | Разделение на вокал и инструментал (2 или 12 стемов) | 12 ₽ |
| `mp4` | Создание музыкального видео | 3 ₽ |
| `midi` | Конвертация в MIDI (после vocal_removal) | Бесплатно |
| `persona` | Создание персоны на основе стиля трека | Бесплатно |

## Примеры запросов

### Генерация музыки

<CodeGroup>
```bash cURL
curl -X POST "https://polza.ai/api/v1/media" \
  -H "Authorization: Bearer <POLZA_AI_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno/generate",
    "input": {
      "prompt": "Энергичный электронный трек для тренировки с драйвовым битом",
      "version": "V5_5"
    }
  }'
```
</CodeGroup>

### Custom Mode (текст + стиль)

<CodeGroup>

  -H "Authorization: Bearer <POLZA_AI_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno/generate",

### Продление трека (операция)

<CodeGroup>
```bash cURL
# Сначала создайте генерацию и получите ID
# Затем используйте audioId из результата генерации

curl -X POST "https://polza.ai/api/v1/media/{generation_id}/operations" \
  -H "Authorization: Bearer <POLZA_AI_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "operation": "extend",
    "params": {
      "audioId": "audio-id-from-generation",
      "prompt": "Продолжение с нарастающим темпом",
      "continueAt": 120
    }
  }'
```
</CodeGroup>

### Upload & Cover

<CodeGroup>
```bash cURL

# 2. Используйте URL для кавер-генерации

```
</CodeGroup>

## Параметры suno/generate

| Параметр | Обязательный | Описание |
|----------|--------------|----------|
| `prompt` | Да | Описание трека или текст песни (до 5 000 символов в custom mode, до 500 в обычном) |
| `style` | Нет | Жанр/стиль музыки (до 1 000 символов). Обязателен в custom mode |
| `title` | Нет | Название трека (до 80 символов). Обязателен в custom mode |
| `version` | Нет | Версия модели: `V5_5` (по умолчанию), `V5`, `V4_5ALL`, `V4_5PLUS`, `V4_5`, `V4`, `V3_5` |
| `customMode` | Нет | Расширенный режим (требует style и title) |
| `instrumental` | Нет | Инструментальная музыка без вокала |
| `negativeTags` | Нет | Стили для исключения |
| `vocalGender` | Нет | Пол вокалиста: `m` или `f` |
| `styleWeight` | Нет | Сила следования стилю (0–1) |
| `weirdnessConstraint` | Нет | Уровень экспериментальности (0–1) |
| `audioWeight` | Нет | Баланс аудиофункций (0–1) |
| `personaId` | Нет | ID персоны для применения стиля |

## Асинхронная генерация

Генерация музыки — асинхронный процесс. После отправки запроса вы получите `id`, который нужно использовать для проверки статуса:

<Steps>
  <Step title="Отправка запроса">
    POST запрос на `/api/v1/media` возвращает `id` генерации
  </Step>

  <Step title="Проверка статуса">
    GET запрос на `/api/v1/media/{id}` возвращает текущий статус
  </Step>

  <Step title="Получение результата">
    Когда статус `completed`, в ответе будут URL аудиофайла, обложка, название и длительность трека
  </Step>
</Steps>
