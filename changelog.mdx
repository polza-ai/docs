## Qwen Image 2.0 и важные исправления для видео и API

### Новая модель: Qwen Image 2.0

Добавлена модель **`qwen2/image-edit`** — редактирование изображений на базе Qwen Image 2.0. Доступна через Media API.

### Расширен лимит загрузки видео

Максимальный размер загружаемого видеофайла увеличен с прежнего до **200 МБ** — актуально для работы с видеогенерацией высокого качества.

### Исправления

- **Восстановлена потеря URL видео** при fallback polling — результат генерации теперь всегда корректно возвращается, даже при переключении между провайдерами
- **Откат лимитов API-ключа** при ошибке резервации баланса — лимиты больше не «сгорают» при неудачных попытках списания
- **Обработка HTTP 422** — ошибки валидации от провайдеров теперь корректно парсятся и возвращаются в читаемом виде
- **Валидация prompt Suno** — в non-custom режиме применяется лимит 500 символов, что предотвращает отклонённые запросы к провайдеру
- **Исправлен парсинг вебхука Точки** — корректно обрабатывается сумма входящего платежа

## Биллинг для бизнеса, новые цены Kling 3.0 и исправления экспорта

### Полноценный биллинг для юридических лиц

Клиенты с бизнес-организациями получили обновлённую систему работы со счетами и платежами:

- **PDF-счёт** с QR-кодом, подписью и печатью — доступен по публичной ссылке для бухгалтерии, хранится в S3 и автоматически удаляется через 30 дней
- **СБП B2B** — оплата счетов по QR-коду прямо с расчётного счёта организации
- **Закрывающие документы (УПД)** — формируются после оплаты, доступны по публичной ссылке
- **Автопополнение баланса** — настройте рекуррентные платежи, чтобы работа не прерывалась
- **Возврат платежей** и отдельная история пополнений вместо общего списка транзакций
- Сквозная нумерация счетов и лимиты сумм пополнения для юр. лиц

## Генерация музыки

Добавлена поддержка генерации музыки через Suno. Создавайте треки с настройкой стиля, вокала, темпа и других параметров через Media API.

## Провайдер BananaLab

Новый провайдер для генерации изображений — NanoBanana Pro (до 4K с точным текстом) и NanoBanana 2 (быстрая генерация). Поддерживается text-to-image и image-to-image.

## Улучшена OpenAI-совместимость

- Поддержка параметров `stop`, `seed`, `n`, `logprobs`, `top_logprobs`, `logit_bias`, `parallel_tool_calls`
- Корректная агрегация множественных вариантов ответа (`n > 1`) в streaming
- Пробрасывание `system_fingerprint` и `logprobs` в ответах
- Аннотации web search (`url_citation`) в streaming и non-streaming режимах
- Исправлены индексы `tool_calls` при параллельных вызовах инструментов

## Хранилище файлов доступно всем

File Storage теперь доступен всем пользователям без ограничений.

## Взвешенная маршрутизация

Трафик между провайдерами теперь распределяется с учётом весов внутри групп приоритета — более точный контроль нагрузки.

## Исправления

- Исправлена завышенная тарификация при `content_filter` ответах
- Исправлен биллинг STT — корректный расчёт по длительности аудио
- Исправлен расчёт стоимости в fallback-цепочке провайдеров
- Исправлен streaming в Responses API — SSE формат и детализация usage
- Исправлено зависание экспорта генераций в статусе processing
- Нормализация CRLF в SSE-парсере для совместимости со всеми провайдерами
- Улучшена надёжность retry и cleanup в media-генерациях
